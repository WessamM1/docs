title:  Create a Service Account
---

## [Overview](#overview)

Follow the steps below to create a [service account](/api-reference/serviceaccount) within your [org](/api-reference/org).

## [Prerequisites](#prerequisites)

- Review the [service account](/api-reference/serviceaccount) reference page
- [Permissions](/api-reference/policy#permissions) to create a [service account](/api-reference/serviceaccount)
- Optional:
  - Install the [CLI](/api-reference/cli)

## [Create using the UI Console](#create-ui)

1. Create a new service account by either:
   - Clicking `Service Accounts` in the left menu and click `New`, or
   - Click the `Create` dropdown in the upper right corner and select `Service Account`
2. Enter a unique name and optional description and click `Next (Tags)`
3. Enter any optional [tags](/api-reference/misc#tags). Click `Create`.
4. Click the `Keys` link to generate a new token that is used to create a profile
5. Enter a key description and click `Add`. Copy and download the generated key

## [Create using the CLI](#create-cli)

Refer to the [serviceaccount create](/api-reference/cli#serviceaccount-create) and [serviceaccount add-key](/api-reference/cli#serviceaccount-add-key) command for details and examples on how to create a service account using the CLI.

## [Next Steps](#next-steps)

After creating a service account with a key, it can be used in a [policy](/api-reference/policy) to grant permissions to any
Control Plane resource. Once permissions have been granted, a [browser-less CLI login](browser-less-cli-login)
is performed to authenticate and allow the execution of CLI commands.
