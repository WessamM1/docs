title:  Create an Audit Context
---

## [Overview](#overview)

Follow the steps below to create an [audit context](/api-reference/auditctx) within your [org](/api-reference/org).

## [Prerequisites](#prerequisites)

- Review the [audit context](/api-reference/auditctx) reference page
- [Permissions](/api-reference/policy#permissions) to create an [audit context](/api-reference/auditctx)
- Optional:
  - Install the [CLI](/api-reference/cli)

## [Create using the UI Console](#create-ui)

1.  Create a new audit context using one of the following methods:
    - Clicking `Audit Contexts` in the left menu and click `New`, or
    - Click the `Create` dropdown in the upper right corner and select `Audit Context`
2.  Enter a unique name and optional description. Click `Next (Tags)`.
3.  Enter any optional [tags](/api-reference/misc#tags). Click `Create`.

## [Create using the CLI](#create-cli)

Refer to the [auditctx create](/api-reference/cli#auditctx-create) command for details and examples on how to create an audit context using the CLI.
